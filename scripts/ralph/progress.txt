## Codebase Patterns
- Auth decorator pattern: `@router.post("/path")` then `@require_auth` below it. Access user via `request.state.auth.user.id`
- Store namespace convention: define as module-level tuple constant e.g. `THREAD_OWNERS_NS: tuple[str, ...] = ("thread_owners",)`
- Store operations: `store.aget(NS, key)` returns item with `.value` dict, `store.aput(NS, key, dict)` to write, `store.asearch(NS, limit=N)` to list, `store.adelete(NS, key)` to remove
- Pydantic request/response models defined at top of router file
- Backend lint: `uvx ruff check .` — no pyright/mypy configured, ruff is the quality gate
- Frontend typecheck: `cd frontend && npx tsc --noEmit` — npm public-hoist-pattern warning is ignorable
- Pre-existing lint errors exist in test files (unused variables) — not related to thread isolation work
- Frontend auth: `useAuth()` from `@/core/auth/AuthProvider` provides `{ user, login, register, logout, refreshUser }`
- Frontend fetch: `fetchWithAuth(url, init)` from `@/core/auth/fetcher` handles credentials/CSRF automatically
- Frontend threads queryKey: `['threads', 'search']` is used by invalidateQueries in multiple places — keep stable
- Sidebar state: `useSidebar().open` from `@/components/ui/sidebar` to check expanded/collapsed
- Data files: `backend/.deer-flow/` (checkpoints.db, memory.json, users.db), `backend/.langgraph_api/` — do NOT delete users.db
