## 示例：Transformer（完整五阶段）

**适用档位：** 🟡 从业者
**核心技巧：** 串行痛点引入、Attention直觉设计、O(n²)工程现实、设计空间（Flash/Sparse/Linear Attention）

**专家快速版（三步）：**
> "Attention 本质是动态权重矩阵：输入 → Q/K/V 投影 → softmax(QKᵀ/√d) · V。
> 设计空间三条主轴：**复杂度**（Dense O(n²) → Sparse → Linear，每降一阶都有表达力代价）、**路由**（全局/局部/交叉，决定信息流拓扑）、**位置编码**（绝对PE → RoPE → ALiBi，影响长度外推）。
> 最大的开放问题：Linear Attention 能否在不损失 in-context learning 的情况下做到真正 O(n)？Mamba/SSM 是一条路但还在追赶。"

**阶段1：问题定义**
> "一句话总结，Transformer想解决RNN无法并行训练的问题。让我们穿越回2017年前，当时训练序列模型就像工厂流水线，必须一个一个字按顺序处理，前一个字没处理完，后面的字只能等着。一个句子有100个字，就要串行100步，GPU的并行算力完全浪费了。"

**阶段2：直觉设计**
> "那你会怎么解决？最直接的想法：能不能让所有字同时看到所有字，然后各自决定'我应该关注哪些字'？比如'银行'这个词，它应该关注前面的'河'还是'钱'来判断意思。如果你这么想，你已经抓住了Attention机制的核心思想。"

**认知分叉（序列模型历史）：**
> "2017年前，处理序列有三条路在竞争：RNN（顺序处理，天然建模时序），CNN（并行卷积，局部感受野），纯 Attention（全局，但当时还没人试过纯注意力做序列）。Transformer 押注了第三条路，并用位置编码补上了序列信息的缺失。RNN 没有死，只是退到了需要严格时序归纳偏置的场景。"

**阶段3：问题迭代**
> "但只有一种关注方式够吗？不够。'银行'可能需要同时关注：词义（河岸还是金融机构）、语法（主语还是宾语）、共指（它指代什么）。那怎么办？弄多个Attention头，每个头学习不同的关注模式。如果你想到这里，你又设计出了Multi-Head Attention。"

**阶段4：工程现实**
> "Attention的计算复杂度是O(n²)。一个句子1000个字，就要算100万次，还要频繁读写显存，慢到怀疑人生。怎么办？
> 1. Flash Attention → 分块计算，减少显存读写，但需要精细的CUDA编程
> 2. Sparse Attention → 不是所有字都要看所有字，但可能丢失长距离依赖
> 3. Linear Attention → 用数学技巧降到O(n)，表达能力稍弱
>
> 这就是工程的艺术：没有银弹。"

**阶段5：升华总结**
> "Transformer的故事告诉我们：有时候打破顺序的束缚，用并行换串行，能带来质的飞跃。这个思想不只在NLP，在CV（ViT）、生物序列（AlphaFold）、时间序列预测里都有应用。核心：让每个元素自己决定关注什么，而不是由架构硬性规定。"

## 示例：贝叶斯定理（简化四阶段）

**适用档位：** 🟢 入门 / 🟡 从业者
**核心技巧：** 医学检测反直觉引入、基率忽视的陷阱、应用迁移

**阶段1：问题**
> "假设你在医院做了个罕见病检测，结果阳性。医生说这个检测的准确率是99%。你会不会觉得自己99%概率得病？答案是：不一定。这就是为什么我们需要贝叶斯定理。"

**阶段2：直觉**
> "直觉上，'检测阳性'有两种可能：你真的得病了（真阳性），或者你没病但检测错了（假阳性）。哪种更大？要看这个病有多罕见。如果这个病只有万分之一的人得，那即使检测99%准确，1万个健康人里也会有100个假阳性，而真阳性只有1个。所以你实际得病的概率只有1/(1+100) ≈ 1%。"

**阶段3：形式化**
> "贝叶斯公式：P(得病|阳性) = P(阳性|得病) × P(得病) / P(阳性)。分子是'得病且检测阳性'的概率，分母是'所有阳性'的概率（包括真阳性和假阳性）。"

**阶段4：应用**
> "贝叶斯定理在哪用？垃圾邮件过滤（这封邮件包含'中奖'，它是垃圾邮件的概率）、医学诊断、机器学习（观测到数据D，模型M是对的概率）。核心：在不确定的世界里，根据新证据更新我们的信念。"
