## Codebase Patterns
- Paths class in `backend/packages/harness/deerflow/config/paths.py` is the central path registry; add new path methods there
- User-scoped paths follow pattern: `{base_dir}/users/{user_id}/...`; thread-scoped: `{base_dir}/threads/{thread_id}/user-data/...`
- Resolve helpers use `user_id=None` fallback pattern: if user_id given â†’ user-scoped, else â†’ sandbox/global path
- `_validate_user_id` and `_validate_thread_id` must be called for any user/thread input used in paths
- Pre-existing test failure: `test_feishu_parser.py` and `test_channels.py::TestFeishuChannel` fail due to missing `app.channels.commands` module (unrelated to this PRD)
- `get_optional_user_id(request)` is the standard way to extract user_id from a FastAPI Request in app/gateway routers
- MemoryMiddleware pattern for extracting user_id: `runtime.context.get('user_id')` first, then `config.metadata` as fallback (guard with try/except RuntimeError for get_config() calls)
- When adding `request: Request` to existing router endpoints, update tests that call the function directly to pass a mock request
- Test lambda mocks for path resolution functions need `_user_id=None` default arg when the function signature gains a user_id param
- Pre-existing test failures in `test_gateway_services.py` (3 tests about build_run_config) are unrelated to this PRD
